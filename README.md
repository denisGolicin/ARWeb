1. **Пазл, собирающийся из картины**
   - *Технологии*: HTML5 Canvas или SVG для отрисовки частей картин в виде пазлов, JavaScript для логики сборки.
   - *Реализация*: Разбить изображение на равные части и перемешать их. Игрок должен перетаскивать части, чтобы собрать оригинальную картину.

2. **Живые картины**
   - *Технологии*: CSS анимации, JavaScript, возможно использование WebGL через библиотеку Three.js для более сложных анимаций.
   - *Реализация простая*: Пользователь наводит смартфон на QR код, затем открывается страница, где пользователь видит аналогичное изображение, но дополненное различными элементами, например, движение облаков, мерцание звезд, движение фигур.
   - *Реализация сложная*: Пользователь наводит смартфон на картину или экспонат, затем открывается страница, где пользователь видит аналогичное изображение, но дополненное различными элементами, например, движение облаков, мерцание звезд, движение фигур.
   - *Реализация невозможная*: Пользователь наводит смартфон на картину или экспонат, и в режиме AR видит экспонат дополненный различными элементами, например, движение облаков, мерцание звезд, движение фигур.

3. **Нахождение лишних предметов на картине**
   - *Технологии*: HTML5 Canvas для отображения изображений и обработки нажатий на определенные области.
   - *Реализация простая*: Пользователь наводит смартфон на QR код, затем открывается страница, где пользователь видит аналогичное изображение с внедренными предметами, не соответствующими контексту, и задача - найти их и нажать на каждый из них.
   - *Реализация сложная*: Пользователь наводит смартфон на картину или экспонат, затем открывается страница, где пользователь видит аналогичное изображение с внедренными предметами, не соответствующими контексту, и задача - найти их и нажать на каждый из них.

4. **Звуковой формат**
   - *Технологии*: HTML5 ```<audio>``` для встраивания звуковых файлов, JavaScript для управления воспроизведением.
   - *Реализация*: Включение фоновой музыки, звуковых эффектов или аудиогидов, которые сопровождают просмотр экспонатов.
  
5. **Игры в формате прыжков по типу Flappy Bird**
   - *Технологии*: Phaser.js или другие HTML5 игровые фреймворки.
   - *Реализация*: Создание простой игры на выживание, где игрок управляет персонажем, прыгающим между препятствиями. Имеется возможность заменить скин персонажа, а также и некоторые элементы заднего фона, чтобы адаптировать внешний вид игры под концепцию выставки или музея.
  
6. **Взаимодействие с виртуальными предметами**
   - *Технологии*: JavaScript для драг-н-дроп интерактивности, возможно использование библиотеки как interact.js.
   - *Реализация простая*: Пользователь наводит смартфон на QR код, затем открывается страница с AR контентом. Пользователь имеет возможность просматривать анимацию, может увеличивать, уменьшать модель. Поддержка форматов .glb для android, и .usdz для iOS.
   - *Реализация сложная*: Позволить пользователям взаимодействовать с предметами на экране, например, собирать виртуальные артефакты в корзину или рюкзак, посредством сканирования экспонатов или QR кодов. Также можно предусмотреть достижения или награды например за сбор всех предметов.
  
7. **Логические загадки с текстом**
   - *Технологии*: HTML и JavaScript для создания интерфейсов вопросов и ответов.
   - *Реализация*: Представление текстовых загадок или викторин с вариантами ответов, связанных с экспонатами или темой выставки.
  
8. **Объемный 3D пазл**
   - *Технологии*: WebGL через Three.js для создания 3D моделей и интерактивности.
   - *Реализация*: Разработка пазла, где части нужно собирать, вращая их в трехмерном пространстве, чтобы воссоздать определенный объект или картину.
  
9. **Механика сражения - для военных музеев**
   - *Технологии*: HTML5 Canvas и JavaScript для создания интерактивных боевых сцен.
   - *Реализация*: Разработка простой механики сражения, где пользователи могут "сражаться" с врагами, кликая по ним или используя элементы стратегии.
  
10. **Кликер по картине с последующим эффектом**
    - *Технологии*: JavaScript и HTML5 для создания кликер-игры.
    - *Реализация*: Пользователь кликает по изображению, и с каждым кликом происходит изменение или появление новых элементов, например, раскрывающиеся детали картины или исторические факты.
